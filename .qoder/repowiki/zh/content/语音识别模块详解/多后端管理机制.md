## 简介
本项目实现了"多后端语音识别管理机制"，通过统一的管理器在浏览器原生 Web Speech API 与阿里云智能语音交互 WebSocket API 之间进行无缝切换。其核心目标包括：
- BackendType 枚举设计与使用：NATIVE 与 ALIYUN 两种后端类型的标识与切换逻辑
- 自动切换算法：基于网络错误检测、失败重试与后端切换条件
- 状态管理策略：SpeechState 枚举的状态及其转换逻辑
- 配置持久化：localStorage 的使用与配置恢复
- 最佳实践与性能对比：为不同网络环境与用户场景提供指导
- 代码示例路径：展示如何在应用中实现灵活的后端管理

**更新** 新增阿里云后端支持，扩展了后端类型枚举和自动切换算法，增强了系统的容错能力和用户体验。

- BackendType 枚举：用于标识当前使用的后端类型（NATIVE / ALIYUN）

### 自动切换算法与失败重试机制
- 触发条件：
  - 原生 API 报错类型为 network 且达到最大重试次数
  - 管理器记录 nativeFailed 标志，提示自动切换到 ALIYUN
- 切换流程：
  - 若已配置阿里云凭证，则自动切换 backend 为 ALIYUN 并启动识别
  - 若未配置，则提示需要在设置中配置阿里云后端
- 失败重试：
  - 原生 API 在非手动停止状态下，每次 end 事件会指数退避重连，最多延时至 2 秒

**更新** 新增阿里云后端的自动切换支持，当原生 API 出现网络错误时，优先检查阿里云配置是否存在，存在则自动切换到阿里云后端。

```mermaid
flowchart TD
Start(["开始识别"]) --> NativeStart["尝试原生识别"]
NativeStart --> NativeOK{"原生成功?"}
NativeOK --> |是| Listen["监听中"]
NativeOK --> |否| NetErr{"网络错误?"}
NetErr --> |否| HandleOther["其他错误处理"]
NetErr --> |是| Retry["增加重试计数"]
Retry --> MaxRetry{"超过最大重试?"}
MaxRetry --> |否| WaitRestart["等待后重试"]
MaxRetry --> |是| HasAliyunCfg{"已配置阿里云?"}
HasAliyunCfg --> |是| SwitchAliyun["切换到阿里云并启动"]
HasAliyunCfg --> |否| Prompt["提示配置阿里云后端"]
WaitRestart --> NativeStart
Listen --> End(["结束"])
SwitchAliyun --> End
Prompt --> End
HandleOther --> End
```

### 状态管理策略（SpeechState）
- 状态定义：
  - IDLE：空闲
  - LISTENING：正在监听
  - ERROR：发生错误
- 状态转换：
  - 原生 start -> LISTENING
  - 原生 end（非手动）-> LISTENING（自动重连）
  - 原生 end（手动）-> IDLE
  - 原生 error -> ERROR（根据错误类型）
  - 阿里云 onStateChange -> 对应映射（idle/listening/error）

**更新** 阿里云后端的状态管理通过 onStateChange 回调函数进行状态同步。

## 性能考量
- 原生 Web Speech API
  - 优点：无需额外网络请求，延迟低；适合国际网络环境
  - 缺点：国内网络可能受限（网络错误），自动重连存在退避延迟
- 阿里云 WebSocket
  - 优点：针对国内网络优化，稳定性较好；支持实时流式识别
  - 缺点：需要网络与鉴权配置；WebSocket 连接与音频编码有额外开销
- 阿里云 WebSocket
  - 优点：提供智能语音交互能力；针对国内网络优化
  - 缺点：需要网络与鉴权配置；WebSocket 连接与音频编码有额外开销
- 最佳实践
  - 默认使用原生后端，遇到网络错误自动切换至阿里云
  - 在设置中预填阿里云凭证，减少首次切换成本
  - 合理设置自动重试上限，避免频繁重试影响体验
  - 使用 UI 状态提示与波形动画增强用户感知

**更新** 阿里云后端提供了更丰富的智能语音交互功能，适合需要高级语音处理能力的应用场景。

## 故障排查指南
- 常见问题与定位
  - 原生网络错误：检查网络连通性与浏览器代理设置；观察自动切换日志
  - 阿里云未配置：在设置面板填写 AppKey/Token；保存后重启识别
  - 权限被拒：检查浏览器麦克风权限；重新授权后重试
  - WebSocket 连接失败：检查网络与 API 配置；查看错误提示
- 日志与提示
  - 原生错误类型会在控制台输出；UI 显示对应错误信息
  - 阿里云错误码与消息会打印到控制台；UI 显示错误状态

**更新** 新增阿里云后端的故障排查指导，包括智能语音交互相关的错误处理。
