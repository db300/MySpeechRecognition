本仓库采用后端统一 JSON 响应、前端状态回调和 Toast 提示的分层错误处理模式，覆盖服务端 Token 获取、浏览器原生 API 调用、阿里云 WebSocket 通信以及 UI 交互四个层面。

## 1. 后端（Node.js / PHP）错误处理
- server.js（Express）：/api/token 路由使用 try/catch 包裹 client.request('CreateToken')，对三类错误分别处理：未配置 AccessKey 返回 { success: false, message }；阿里云返回 ErrCode（如 40020503 无权限）构造友好中文消息并返回 500；网络或 SDK 异常 catch 后同样返回 500 JSON。
- api/token.php（PHP）：手动实现 HMAC-SHA1 签名，对 curl 错误、JSON 解析失败、ErrCode 分支均输出相同结构的 JSON，保证 Node 与 PHP 两个后端接口契约一致。
约定：所有错误均以 { success: boolean, message?: string, ... } 形式返回，不抛 HTTP 异常或裸文本。

## 2. 前端应用层（app.js）
- _autoGetToken() 依次尝试 /api/token 和 /api/token.php，通过 fetch + try/catch 逐个兜底，任一成功即填充 token；全部失败则按钮置 error 态并通过 _showToast 提示。
- _onStateChange(state, message) 根据 SpeechState.ERROR 更新 status 元素样式为 .error，并在特定场景自动打开设置面板引导用户修复配置。
- 所有用户可见的错误信息都经过 _showToast(message) 以 2 秒动画弹出，避免直接 alert。

## 3. 语音识别引擎层（aliyun-speech.js）
- 配置校验：startListening 开头检查 isConfigured()，缺失 AppKey/Token 时通过 stateChangeCallback('error', ...) 上报。
- 浏览器 API 错误分类：捕获 NotAllowedError/PermissionDeniedError（麦克风拒绝）、NotFoundError（无设备）、含 WebSocket 的异常，分别映射到不同中文提示。
- WebSocket 生命周期：onerror 在连接建立前 reject Promise，建立后交由 onclose 处理；onclose 区分从未建立成功和已建立后断开，后者触发 _tryReconnect()；重连策略指数退避（1s、2s、3s上限 5s），最多 3 次，超过则停止并上报 error。
- 业务错误码：_onMessage 中解析 header.status，当大于等于 40000000 且小于 50000000 时视为客户端错误，立即清理资源并回调 error。

## 4. 设计决策与约束
- 不使用自定义 Error 类或全局中间件，而是以结构化 JSON + 回调/Toast 贯穿全链路，保持单进程 Express + 静态页面的轻量架构。
- 错误传播路径固定为：底层异常到上层 catch 包装成用户可读 message再到通过回调或 UI 反馈，禁止将原始堆栈暴露给前端。
- 对外部依赖（阿里云 SDK、curl、WebSocket）的错误一律降级为可恢复的用户提示，而非让进程崩溃。
